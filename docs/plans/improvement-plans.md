## В работе / Следующие шаги

### 3. Paper titles как vocabulary (вместо dictionary-terms)

**Проблема:** Текущий словарь — абстрактные термины ("anomaly detection", "clustering"). Декодированные концепты безликие, не дают направления для идеи.

**Решение:** Заменить vocabulary на реальные названия научных статей из открытых API. Потомок будет декодироваться не в "neural network + transfer learning", а в конкретные paper titles — сразу видно исследовательскую территорию.

**Источники (все бесплатные):**
- **Semantic Scholar** (рекомендуется) — REST API, 100 req/sec без ключа, хороший CS coverage
- **arXiv** — бесплатно, но менее удобный API
- **OpenAlex** — полностью открытый, 250M работ

**Что нужно:**
1. Скрипт для скачивания ~5-10K titles по нашим тематикам
2. Пересчитать эмбеддинги (`generate-vocabulary-embeddings.mjs` уже умеет)
3. Адаптировать `decodeConcepts` для длинных titles
4. Адаптировать `generateOffspringText` — titles уже осмысленные фразы

**Оценка:** ~2-3 часа, $0

### 4. 3-parent crossover

**Проблема:** Два родителя дают интерполяцию на линии A↔B. Потомки ограничены отрезком между двумя точками в пространстве — все идеи "между" родителями.

**Решение:** Добавить третьего родителя. Три точки задают плоскость — экспоненциально больше пространства для потомков. Третий родитель вытягивает ребёнка из "очевидной середины" между A и B, генерируя неожиданные гипотезы.

**Что нужно:**
1. Расширить `crossover()` для 3 родителей (взвешенная сумма 3 векторов, 6 блоков × 3 веса)
2. Расширить `generateOffspring()` для опционального третьего родителя
3. UI: третий слот для Parent C в LabPage + дополнительные слайдеры весов
4. `computeFitness`: учесть balance для трёх родителей
5. Connections: child-of для трёх родителей при promote

**Оценка:** ~3-4 часа, $0

**Комбинация 3 + 4:** paper titles делают выход осмысленным, 3-parent crossover делает вход разнообразным. По отдельности каждое решает только половину проблемы.
